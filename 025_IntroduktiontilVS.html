    Introduktion til Visual Studio

    Om modulet

    Dette modul vil introducere dig til Visual Studio og debuggeren.

    Installation

    Visual Studio kan findes på

    https://visualstudio.microsoft.com/

    og findes som tidligere nævnt i flere versioner. Community Edition er den gratis version (for private og mindre firmaer - se licens betingelser).

    Den kan installeres på Windows og Mac, og du skal i installationsproceduren mindst vælge “.NET desktop development”.

    Version

    Den sidste nye version af Visual Studio er VS 2022 fra november 2021.

    Oprettelse af et projekt

    Man kan skabe mange forskelige projekttyper i Visual Studio, men vi vil her udelukkende fokusere på en “Console Application (.NET Core)” for på den måde at holde fokus på koden og ikke på brugerflade. Så kan du efterfølgende selv lege med andre projekttyper.

    Du kan skabe en ny applikationer ved at vælge File->Project og så vælge C# i “Language” dropdown-boksen, og søge efter console. Når du først har brugt den en gang dukker projekttypen op i listen til venstre i vinduet.

    Herefter kan man angive navnet på projektet og hvilken mappe det skal placeres i.

    En C# løsning

    En C# løsning (solution) består typisk af et eller flere projekter, og et enkelt projekt kan være en konsol applikation, web applikation, windows applikation, mobil applikation med videre. En løsning er repræsenteret ved en .sln fil.

    Hvert projekt kan så være opdelt i flere filer, som kompileren vil sørge for at kompilere sammen. Et projekt er repræsenteret af en .csproj fil, og består typisk af en masse .cs filer med kode samt eventuelle konfigurationsfiler.

    Når man skabe en konsol applikation består den altså af en .sln-fil (løsning)

    og en .csproj-fil (projekt) samt de filer som projektet består af.

    I Visual Studio kan finde løsning, projekter og filer i “Solution Explorer”:

    Af andre vigtige vinduer i VS kan nævnes “Error List” vinduet hvor eventuelle kompileringsfejl dukker op.

    Mange kodefiler i et projekt

    Man må gerne have mange cs-filer (kodefiler) i et projekt, og kan organisere disse som man ønsker i mapper. Alle filer skal dog bestå af en typedefinition (klasse, struktur, enum med videre). Man kan sågår i den mere avancerecede C# fortælle kompileren at den skal kombinere typer. Det kan ske ved hjælp af partial kodeordet - herom senere.

    Man kan også vælge at have flere typer i samme fil - kompilereren er ligeglad.

    Et konsol projekt med flere typer kan altså enten består af

    • en stor program.cs med mange typer definitioner

    eller

    • en program.cs og en fil for hver type (hund.cs, faktura.cs mv)

    eller

    • en program.cs og mapper (højreklik på projekt og “New folder”) med en fil for hver type
      • \regnskab
        • faktura.cs
        • fakturalinje.cs
      • \personale
        • person.cs
        • medarbejder.cs

    eller en kombination.

    IntelliSense

    Når du skriver kode i Visual Studio så lær at bruge “IntelliSense” - altså at lade Visual Studio skrive så meget af kode som muligt ved at vælge på listen der automatisk dukker op. Det er en af fordelene ved at arbejde med et typestærkt sprog som C# - VS ved hvad en konkret type indeholder.

    Hold øje med listen når du skriver, og tryk Enter eller Tab når det rigtige er valgt på listen.

    Prøv den måde kan du nemt se havd en type består af, og staver ikke forkert.

    IntelliCode

    Med VS 2022 følger (normalt) Microsofts IntelliCode extension som udvider hjælpen fsva syntaks og instruktioner med AI. Hvis du ønsker denne extension kan den disables - alternativt tilrettes under options.

    Snippets

    Lad VS gøre så meget af arbejdet som overhovedet muligt - brug kode snippets så meget du kan. Der findes en del indbyggede, og du kan oprette dine egne.

    Find de forskellige snippets ved at højre klikke og vælge Snippet->Insert Snippets->Visual C#

    Du kan også bruge genvejsnavnet og blot trykke 2 gange tabulering

    Her er nogle af de mere kendte snippets til C#

    • for = for loop
    • try = try catch
    • do = do/while
    • if = if
    • cw = Console.WriteLine()

    Genvejstaster

    VS består af et hav af genvejstaster som du efterhåndende vil lære - her er nogle af de vigtigste så du har en liste at komme retur til:

    • Ctrl+K+D = Format code
    • Ctrl+K+C = Comment selection
    • Ctrl+K+U = Uncomment selection
    • Ctrl+K+X = Insert snippet
    • Ctrl+K+S = Surround with (snippet)
    • F12 = Goto definition
    • Ctrl+K+R = Find all references
    • Ctrl+. = Quick actions
    • Ctrl+Space = Activate intelliSense
    • F5 = Start with debugger
    • Ctrl+F5 = Start without debugger
    • Shift+F5 = End debugger session
    • F9 = Set breakpoint
    • F10 = Step over
    • F11 = Step into
    • Ctrl+Shift+F9 = Delete all breakpoints
    • Alt+Up+Down = Move selected code
    • Ctrl+Shift+B = Build
    • Ctrl+, = Global find types, members, files
    • Ctrl+Tab = switch between windows
    • F2 = rename

    Vi kommer tilbage til flere af dem senere.

    Hjælp fra Visual Studio

    Når du koder vil Visual Studio forsøge at hjælpe dig så meget den kan. Det sker eksempelvis ved hjælp af et ikon af en pære.

    Det kan også ske ved hjælp af et ikon af en skruenøgle.

    Tastaturgenvejen til hjælp er Ctrl + punktum.

    I starten vil du nok ikke benytte den hjælp Visual Studio tilbyder, men efterhånden som du bliver mere erfaren så prøv at være lidt nysgerrig, og lad Visual Studio ændre koden. Det er en god måde at blive lidt klogere på C# syntaks, og du kan altid fortryde en ændring.

    Kompilering

    Et projekt kan kompileres (bygges) på flere måder

    • Vælg “Build [projekt]” på Build-menuen
    • Med fokus i en kodefil kan du trykke F6
    • Ctrl+Shift+B

    Under alle omstændigheder skulle det gerne ende med en “Build succeeded” i venstre nederste hjørne.

    Applikationen kan herefter afvikles fra konsolen (eller flyttes til server/mobil).

    Afvikling

    En applikation kan også bygges og afvikles direkte fra VS

    • Debug->Start debugging
    • F5

    eller

    • Debug->Start without debugging
    • Ctrl+F5

    Typisk vil man starte med debuggeren, men mere om det senere.

    Simpel debugging

    Video Simpel debugging (15 min.)

    Benchmark

    En af de mere avancerede profiling pakker

    • Benchmark.net
    // Package 'BenchmarkDotNet'
using System.Threading;
using BenchmarkDotNet.Attributes;
using BenchmarkDotNet.Running;

namespace MyBenchmarks
{
    public class Program
    {
        public static void Main(string[] args)
        {
            var summary = BenchmarkRunner.Run(typeof(IntroBasic).Assembly);
        }
    }

    public class IntroBasic
    {
        [Benchmark]
        public void Sleep() => Thread.Sleep(10);

        [Benchmark(Description = "Thread.Sleep(10)")]
        public void SleepWithDescription() => Thread.Sleep(10);
    }
}
