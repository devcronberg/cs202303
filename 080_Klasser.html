    Klasser

    Om modulet

    I dette modul vil du lære den helt grundlæggende teori bag klasser. Du skal have en forståelse for de forskellige typer af medlemmer du kan bruge til at designe dine klasser, og øve dig i at kunne finde entiteter i en applikation, og abstrahere disse til klasser. Dem kan du så bruge til at skabe objekter af.

    Der er en del opgaver i dette modul som du bør bruge lidt tid på.

    Klasse er en skabelon for instanser (typisk kaldet objekter)

    • En skabelon kan bruges til mange objekter
    • Den eneste forskel på objekter af samme type er data (tilstand)

    Oprettelse af objekter

    Video Oprettelse af objekter (4 min.)

    Se video

    Når man skal oprette et objekt af en klasse benytter man new-kodeordet, og referencen skal gemmes i en variabel af den korrekte type. HUSK - at klasser er en reference baseret variabel.

    // variabel som kan pege på et objekt
Klasser1.Person p1;
// Nyt objekt på heap og bind ref til p1
p1 = new Klasser1.Person();

// eller på samme linje
Klasser1.Person p2 = new Klasser1.Person();
// men p2 er stadig en variabel der kan pege på et Person-objekt

// derfor kan man
p1 = p2;
// hvorefter det p1 tidligere pegede på er væk


// Fra C# 9 kan man også

Klasser1.Person p3 = new ();

    Her er en kort animieret gif - en gul linje viser at der skal til at blive afviklet! Du skal være helt skarp på hvad der sker:

    • En variabel på stack’en (del af hukommelse) kan pege på et objekt på heap (anden del af hukommelse)
    • new-kodet skaber objektet på heap’en
    • = gemmer referencen til objektet i variablen
    • Hvis et objekt på heap’en ikke er refereret bliver det automatisk fjernet

    System.Object

    Video System.Object (3 min.)

    Se video

    Alle referencebaserede typer arver fra System.Object, og alle værdibaserede typer er kompatible med System.ValueType, som arver fra System.Object.

    Derfor består alle typer af metoderne:

    Navn Beskrivelse
    (virtual) bool Equals Sammenligner to instanser
    (virtual) int GetHashCode Skaber en hash værdi til gruppering
    Type GetType Returnerer typen
    (virtual) string ToString Konverterer instans til en streng 
    int a = 1;
Console.WriteLine(a.Equals(1));
Console.WriteLine(a.GetHashCode());
Console.WriteLine(a.GetType().Name);
Console.WriteLine(a.ToString());

System.IO.FileInfo b = new System.IO.FileInfo(@"c:\temp\test.txt");
System.IO.FileInfo c = b;
Console.WriteLine(b.Equals(c));
Console.WriteLine(b.GetHashCode());
Console.WriteLine(b.GetType().Name);
Console.WriteLine(b.ToString());

    Så hvis du undre dig over hvor eksempelvis ToString-metoden kommer fra på en ellers “tom” klasse er det derfor.

    Felter

    Video Felter (6 min.)

    Se video

    • Felter (fields) er objekternes data
    • kan være alle typer

    • Synlighed
    • private (default)
      • Kan kun tilgås internt
    • public
      • Kan tilgås både internt og eksternt
    namespace Klasser2
{
    public class Person
    {
        public string navn;
        private int alder;
        private Køn køn;

        public void Test() {
            // her kan alle felter tilgås - både private og public
        }
    }

    public enum Køn {
        Mand,
        Kvinde
    }
}
    • Felter tilgås med punktumnotation
    Klasser2.Person p1 = new Klasser2.Person();
p1.navn = "Mikkel";
Console.WriteLine(p1.navn);
// p1.alder = 15; FEJL : alder kan ikke tilgås - den er privat for klassen
// Console.WriteLine(p1.alder);  Fejl: alder kan ikke tilgås -den er privat for klassen

/*
 ---------- Output: ----------

Mikkel

    readonly felter

    • Et readonly felt kan kun intialiseres i constructor eller ved intialisering og kan derefter ikke tilrettes
    using System;

namespace Klasser14
{
    public class Person
    {
        public readonly string Navn = "";

        public Person(string navn)
        {
            this.Navn = navn;
        }

        public void Test() {
            // this.Navn = ""; fejl - må kun tildeles i constructor eller ved initialisering
        }
    }
}

    Det giver mulighed for at skabe immutable felter.

    Privat constructor

    • Hvis man tilføjer en privat default constructor kan man ikke skabe et objekt af klassen
    • Singleton-mønster
    namespace Eksempler
{
    public class Singleton
    {
        // Man må ikke skabe en instans - kun HentSingleton er mulig
        private Singleton() { }

        // instans data (bør ikke være offentligt felt)
        public int Id;
        // mv...

        private static Singleton o = null;

        public static Singleton HentSingleton() {
            if (o == null)
                o = new Singleton();
            return o;
        }

    }
}

    Custom constructor

    Video Custom constructor (10 min.)

    Se video

    • Metode(r) der automatisk afvikles ved brug af new
    • Har argumenter
    • Bruges typisk til initialisering af private felter
    • Man kan nøjes med en custom eller flere custom constructor’s
    namespace Klasser4
{
    public class Person
    {
        private string navn;
        private int alder;

        public Person(string navn, int alder)
        {
            this.navn = navn;
            this.alder = alder;
        }

        // Man må gerne have flere
        public Person(string navn)
        {
            this.navn = navn;
            this.alder = 0;
        }

    }
}
    • Så skal man benytte den/dem ved oprettelse af objekt
    // Brug af custom construtor
Klasser4.Person p1 = new Klasser4.Person("Mikkel", 15);
Klasser4.Person p2 = new Klasser4.Person("Mikkel");
// Klasser4.Person p1 = new Klasser4.Person(); Fejl: Default constructor findes ikke
    • Du kan vælge både at have en default og custom constructor
    namespace Klasser5
{
    public class Person
    {
        private string navn;
        private int alder;

        public Person()
        {
            this.navn = "";
            this.alder = 0;
        }

        public Person(string navn, int alder)
        {
            this.navn = navn;
            this.alder = alder;
        }

        // Man må gerne have flere
        public Person(string navn)
        {
            this.navn = navn;
            this.alder = 0;
        }

    }
}
    • Så kan du frit vælge ved oprettelse hvilken constructor du vil benytte
    // Brug af forskellige constructors
Klasser5.Person p1 = new Klasser5.Person();
Klasser5.Person p2 = new Klasser5.Person("Mikkel");
Klasser5.Person p3 = new Klasser5.Person("Mikkel", 15);

    Genbrug af constructor

    Video Genbrug af constructor (10 min.)

    Se video

    • Man kan binde flere constructors sammen for at undgå kopi af (initaliserings- og validerings)kode
    • Brug af this-kodeordet
    namespace Klasser6
{
    public class Person
    {
        private string navn;
        private int alder;

        // "mor" for alle constructors
        public Person(string navn, int alder)
        {
            this.navn = navn;
            this.alder = alder;
        }

        public Person() : this("", 0)
        {
        }

        public Person(string navn) : this(navn, 0)
        {
        }

    }
}

    Statiske medlemmer

    Video Statiske medlemmer (9 min.)

    Se video

    • Ligesom diverse typer har instans og statiske metoder:
    int i = 10;
// Instans metode
Console.WriteLine(i.ToString());
// Statisk metode
Console.WriteLine(int.MinValue);


/*
 ---------- Output: ----------

10
-2147483648

*/
    • kan klasser også have statiske medlemmer - typisk metoder (men egenskaber og felter kan også være statiske).
    • giver mulighed for at skabe medlemmer som kan benyttes uden at skabe et objekt
    • klasser kan også markeres som statiske (må kun indeholde statiske metoder)
    namespace Klasser12
{
    public class Person
    {
        // instans medlemmer
        public string Navn { get; set; }
        public string StortNavn() {
            return this.Navn.ToUpper();
        }

        public static string KonverterTilStortNavn(string navn) {
            return navn.ToUpper();
        }

    }

}
    • brug af klasse
    Klasser12.Person p = new Klasser12.Person() { Navn = "mikkel" };
Console.WriteLine(p.StortNavn());
Console.WriteLine(Klasser12.Person.KonverterTilStortNavn("mikkel"));

/*
 ---------- Output: ----------

MIKKEL
MIKKEL

*/
    • Man kan også benytte statiske data og en evt tilhørende statisk constructor
    namespace Klasser12
{
    public class Vare
    {
        // Konstant
        // private static double momsPct = 0.25;
        // eller
        private static double momsPct = 0;

        static Vare() {
            momsPct = .25;  // Kan komme fra en database
        }
    }

}
    • Masser af medlemmer i frameworket er statiske
    double res = System.Math.Cos(10.3);
bool f = System.IO.File.Exists("c:\\temp\\test.txt");

    Attributter

    For at give yderligere information til eksempelvis kompiler eller frameworks som benytter en klasse kan den beriges med [attributter]. Det vil tilføjes ekstra metadata som kan aflæses hvis det ønskes. Eksempelvis kan en klasse markeres med en SerializableAttribute for at indikere, at klassen kan benyttes i serialisering (mest brugt i ældre kode):

    [Serializable()]
public class Person
{
    // ...
}

    Der må gerne tilføjes mange attributter og man kan skabe dem selv, men lige nu skal du bare vide at det er en måde at tilføje ekstra information til klasser.

    Klassediagram

    I Visual Studio har du mulighed for at indsætte et klasse diagram der viser hvad en klasse består af og eventuelt sammenhæng med andre klasser.

    Du kan indsætte et klassediagram ved at højreklikke på projektet, vælge “Add” og “New item…” - her skal du lede efter “Class Diagram”.

    Hvis du ikke kan finde klassediagrammet er det muligvis din installation mangler et kryds i en checkboks. Åbn Visual Studio Installer, vælg Modify, og under “Individual components” skal du finde “Class designer” og sætte kryds i checkboksen.

    Nullable reference types

    I C# 8 kan man anmode kompileren om at hjælpe med at finde mulige null reference exceptions. Det kan slås til på flere måder:

    <Project Sdk="Microsoft.NET.Sdk">
    <PropertyGroup>
        <OutputType>Exe</OutputType>        
        <TargetFramework>net6.0</TargetFramework>
        <Nullable>enable</Nullable>
        <WarningsAsErrors>Nullable</WarningsAsErrors>
    </PropertyGroup>
</Project>

    eller direkte i kode

    #nullable enable
//
#nullable restore

    Nu vil eksempelvis følgende klasse skabe en warning omkring Navn

    public class Hund
{
  // Non-nullable property 'Navn' is uninitialized. Consider declaring the property as nullable.
    public string Navn;
    public int Alder;

    public string NavnMedStort()
    {
        return this.Navn.ToUpper();
    }
}

    Warning kan fixes på flere måder - initialisering med “” i erklæring eller constructor

    public class Hund
{
    public string Navn;
    public int Alder;
    public Hund()
    {
        Navn = string.Empty;
    }
    public string NavnMedStort()
    {
        return this.Navn.ToUpper();
    }
}

    Hvis du efterfølgende forsøger at tildele Navn værdien null, vil der dukke en ny warning op.

    Hund d = new Hund();

// Cannot convert null literal to non-nullable reference type.
d.Navn = null;

    Du kan eventuelt markere Navn som en null reference type, men nu vil der dukke en ny warning op.

    public class Hund
{
    public string? Navn;
    public int Alder;
    public Hund()
    {
        Navn = string.Empty;
    }
    public string NavnMedStort()
    {
        // Dereference of a possibly null reference
        return this.Navn.ToUpper();
    }
}

    Den må man så tage stilling og eventuelt fixe som:

    public class Hund
{
    public string? Navn;
    public int Alder;

    public Hund()
    {
        Navn = string.Empty;
    }

    public string NavnMedStort()
    {
        if (this.Navn == null)
            throw new ApplicationException("Fejl i data");
        return this.Navn.ToUpper();
        // eller noget ala
        // return this.Navn!.ToUpper();
        // return this.Navn == null ? "" : this.Navn.ToUpper();
    }
}

    Her er ! en ny operator der fortæller kompileren at den skal blande sig uden om.

    Boxing / Unboxing

    • Boxing

    • gem værdibaseret type i en reference baseret type

    • Unboxing

    • hent en værdibaseret type fra en reference baseret type

    • kan skabe runtime fejl på runtime (kompiler kan ikke se fejl)

    • perfomance
    • brug generics
    int i = 10;
Console.WriteLine(i);
// boxing
object o = i;
Console.WriteLine(o);

int x = Convert.ToInt32(o);
Console.WriteLine(x);


/*
 ---------- Output: ----------

10
10
10

*/
