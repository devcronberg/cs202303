    Expression trees

    Om modulet

    Dette modul vil du lære hvordan du kan arbejde med expression trees i C#.

    Om Expression trees

    Expression trees giver mulighed for at konvertere kode til data. Det udnyttes blandt andet LINQ-frameworks, AutoMapper med videre.

    Kan også sige, at et EXT (expression tree) i C# svarer til et ATS (Abstract Syntax Tree) relateret til en compiler - men EXT repræsenterer et enkelt udtryk hvor AST repræsenterer et program.

    a + b * 2

    [+]
  /    \
 a     [*]
      /   \
     b     2
    Ref

    Func<int, bool> a = i => i > 5;
Console.WriteLine(a(3));                    // false

Expression<Func<int, bool>> b = i => i > 5;
Console.WriteLine(b);                       // i=> (i > 5);
Console.WriteLine(b.GetType());             // System.Linq.Expressions.Expression<System.Int32,System.Boolean>            
Console.WriteLine(b.Type);                  // Func<int, bool>
Console.WriteLine(b.Parameters.GetType());  // ReadOnlyCollection<System.Linq.Expressions.ParameterExpression>
Console.WriteLine(b.Parameters[0].Name);    // i
Console.WriteLine(b.Parameters[0].Type);    // Int32
Console.WriteLine(b.Body);                  // (i > 5);
Console.WriteLine(b.Body.GetType());        // System.Linq.Expressions.LogicalBinaryExpression

BinaryExpression b1 = (BinaryExpression)b.Body;
Console.WriteLine(b1.Left);                 // i
Console.WriteLine(b1.Right);                // 5
Console.WriteLine(b1.NodeType);             // GreaterThan

Func<int, bool> c = b.Compile();
Console.WriteLine(c(3));                     // false

Expression<Func<int, int, int>> d = (x, y) => x > 10 ? x + y : x - y;
Console.WriteLine(d);                       // (x, y) => IIF((x > 10), (x + y), (x - y))
Console.WriteLine(d.Body.GetType());        // System.Linq.Expressions.FullConditionalExpression

Expression<Func<string, string>> e = x => x.ToUpper();
Console.WriteLine(e);                       // x => x.ToUpper()
Console.WriteLine(e.Body.GetType());        // System.Linq.Expressions.InstanceMethodCallExpression

    Expression API

    Der findes en masse typer til at beskrive et træ, og mange kan oprettes gennem statiske metoder på Expression-klassen.

    Parametre kan beskrives med en ParameterExpression, og Body med en LambdaExpression eller en BinaryExpression. I selve body’en kan der benytte ConstantExpression, MethodCallExpression, PropertyExpression (meget benyttet i EF, MVC mv) samt et hav af operatorer (LessThan, Equal, GreaterThan mv).

    Func<string, string> mål = x => x.ToUpper();

var p = Expression.Parameter(typeof(string));
var t = typeof(string).GetMethod("ToUpper", Type.EmptyTypes);
var b = Expression.Call(p, t);
var l1 = Expression.Lambda(b, p);
var e1 = l1.Compile();
Console.WriteLine(e1.DynamicInvoke("a"));
var l2 = Expression.Lambda<Func<string, string>>(b, p);
var e2 = l2.Compile();
Console.WriteLine(e2("a"));

    IQueryable

    Hvor IEnumerable arbejder på samlinger

    public interface IEnumerable
{
    IEnumerator GetEnumerator();
}

    arbejder IQueryable på en Expression

    public interface IQueryable : IEnumerable
{
    Type ElementType { get; }
    Expression Expression { get; }
    IQueryProvider Provider { get; }
}

    Eksempelvis er Where-metoden på Linq to objects og Linq to EF fundamentalt forskellige fra hinanden:

    public static IEnumerable<TSource> Where<TSource>(this IEnumerable<TSource> source, Func<TSource, int, bool> predicate);

public static IQueryable<TSource> Where<TSource>(this IQueryable<TSource> source, Expression<Func<TSource, bool>> predicate);

    designtime compile -> runtime compile

    Hvorfor benytte expressions:

    // Lambda
Func<int, int, int> a = (x, y) => x + y;
Console.WriteLine(a(2, 2));   // 4

// Expression
Expression<Func<int, int, int>> b = (x, y) => x + y;
Console.WriteLine(b.Compile()(2, 2)); // 4

// Expression API
var p1 = Expression.Parameter(typeof(int), "x");
var p2 = Expression.Parameter(typeof(int), "y");
var add = Expression.Add(p1, p2);
var exp = Expression.Lambda(add, p1, p2).Compile();
Console.WriteLine(exp.DynamicInvoke(2, 2)); // 4

// Scripting (Microsoft.CodeAnalysis.CSharp.Scripting)
string expString = "(x, y) => x + y";
Func<int, int, int> s = CSharpScript.EvaluateAsync<Func<int, int, int>>(expString).Result;
Console.WriteLine(s(2, 2)); // 4
