    Hvorfor metoder

    // Gentaget kode (ikke godt... - svært at vedligeholde og overskue)
for (int i = 1; i < 6; i++)
    Console.WriteLine(i);


for (int i = 1; i < 6; i++)
    Console.WriteLine(i);


for (int i = 1; i < 6; i++)
    Console.WriteLine(i);

// Løkker i løkker (bedre ud - men kan ikke genbruges)
Console.WriteLine();
for (int x = 0; x < 3; x++)
    for (int i = 1; i < 6; i++)
        Console.WriteLine(i);

// Goto (bedre, nu kan det genbruges - men igen - svært at overskue)
Console.WriteLine();
int programLinjeTæller = 0;
start:
if (programLinjeTæller < 3)
    goto tæl;
else
    goto slut;
tæl:
for (int i = 1; i < 6; i++)
    Console.WriteLine(i);
programLinjeTæller++;
goto start;
slut:

// metoder (meget bedre - nemt at genbruge og kode lever i eget virkefelt)
Console.WriteLine();
for (int x = 0; x < 3; x++)
{
    Tæl();
}
void Tæl()  // kan nemt udvides til at tælle til et vilkårligt antal
{
    for (int i = 1; i < 6; i++)
        Console.WriteLine(i);
}

    Den traditionelle visuelle forklaring

    En metode kan ses som en boks med instuktioner. På venstre sider findes eventuelle værdier (argumenter) som sendes ind i boksen, og på højre side en eventuel værdi som kan hentes ud af boksen. Boksen med instruktioner er typisk grå eller sort for at indikere, at indholdet (instruktionerne) ikke er vigtige. Det er argumenter og returværdi der skal være fokus på når en metode kaldes.

    Metoder uden returværdi hedder i C# en void-metode (void kan oversættes til tom).

    Programpointeren

    F1();
void F1() {
    Console.WriteLine("I F1()");
    F2();
}
void F2() {
    Console.WriteLine("I F2()");
    F3();
}
void F3() {
    Console.WriteLine("I F3()");
    F4();
}
void F4() {
    Console.WriteLine("I F4()");
    F5();
}
void F5() {
    Console.WriteLine("I F5()");
    // F1(); (puha - skaber rekursivt kald)
}

    Hvor skal metoder placeres

    • Alt i C# er placeret i enten klasser (class) eller strukturer (struct)
    • Metoder kan enten være instans- eller statiske metoder
      • Se senere
    class TestKlasse
{
   public void TestMetode1() { }
   public static void TestMetode2() { }

}

struct TestStruktur
{
   public void TestMetode1() { }
   public static void TestMetode2() { }

}

    Metoderne kan nu kaldes som

    // brug af instans metoder
TestKlasse t1 = new TestKlasse();
t1.TestMetode1();
TestStruktur t2 = new TestStruktur();
t2.TestMetode1();

// brug af statiske metoder
TestKlasse.TestMetode2();
TestStruktur.TestMetode2();

    Hvis man arbejder med konsol-applikationer arbejder man jo allerede i en klasse (Program), og Main-metoden er i sig selv statisk. Derfor er den nemmeste måde at teste metoder på, at skabe statiske metoder i Program-klassen.

    namespace MinKonsolApp
{
    class Program
    {
        static void Main(string[] args)
        {

            // kald af metode
            TestMetode();

        }

        static void TestMetode() { }
    }
}

    Synlighed

    • public
      • Kan tilgås både internt og eksternt
    • private
      • Kan kun tilgås internt
    • (protected mv)

    Denne klasse viser både offentlige og private metoder:

    public class Person
{
   public string Navn;
   public int Alder;

   // offentlige metoder
   public string NavnMedStort()
   {
       return this.Navn.ToUpper();
   }

   public void GemPerson()
   {
       // Offentlige metoder kan godt kalde
       // private metoder
       CheckPerson();
       // kode til at gemme person
   }

   // private metoder
   private void CheckPerson()
   {
       if (Alder < 0 || Alder > 100 || Navn == null || Navn == "")
           throw new ApplicationException("Forkerte data");
   }
}

    Metoder i metoder

    I nyere version af C# er det muligt at placere metoder i metoder.

    void Main(string[] args)
{

    int res = LægSammen(1, 1);

    int LægSammen(int a, int b)
    {
        return a + b;
    }

}

    Lidt mere avanceret (forudsætter viden om delegates og lambda)

    // brug af anonyme metoder
// Lambda - se senere

Func<int, int, int> LægSammen = (a, b) => a + b;
int res = LægSammen(1, 1);
Console.WriteLine($"Resultat: {res}");


/*
 ---------- Output: ----------

Resultat: 2

*/

    Introduktion til Expression Body methods

    Fra og med C# 6 har det været muligt at benytte såkaldte expression body members - herunder til definering af metoder der kun har en instruktion (med en implicit return). Den benytter lambda syntaksen (som gennemgås senere), men giver mulighed for at skrive metoder meget hurtigt. Her er et par eksempler på brug af syntaksen:

    private static double Gange(double a, double b) => a * b;

    er det samme som

    private static double Gange(double a, double b)
{
    return  a * b;
}

    og

    private static void Skriv(string t) => Console.WriteLine(t);

    er det samme som

    private static void Skriv(string t)
{
     Console.WriteLine(t);
}

    Som sagt - mere om Lambda i et senere modul.

    Rekursive metoder

void RekusivMetode(int start, int stop) {
    Console.WriteLine(start);
    if (start == stop)
        return;
    start++;
    RekusivMetode(start, stop);
}

    Brug af in, ref og out ved metoder

    • ref sender reference med til værdibaserede variabler
      • de skal være initialiserede
    int i = 10;
Console.WriteLine(i);
Metode(ref i);
Console.WriteLine(i);

void Metode(ref int a) {
    a++;
}

/*
 ---------- Output: ----------

10
11

*/
    • out sender reference med til værdibaserede variabler
      • de behøver ikke være initialiserede
    int i;
Metode(out i);
Console.WriteLine(i);

void Metode(out int a)
{
    a=10;
}

/*
 ---------- Output: ----------

10

*/

    Ref ved reference variabler

    • Ved reference baserede argumenter kan man ikke ikke rette referencen (eksempelvis pege på et nyt objekt)
    • Men ref give mulighed for helt at ændre referencen
    void Test1(Person p)
{
    // der er overført en kopi af p
    // så hvis man tildeler p en ny
    // reference har det ikke nogen
    // effekt i den kaldende metode
    p = new Person();   // ingen betydning
}
void Test2(ref Person p)
{
    // der er overført den originale reference
    // så hvis man tildeler p en ny reference har
    // det betydning i den kaldende metode
    p = new Person();   // betydning
}

    in

    in kan bruges ved structs til at sikre, at værdien ikke ændres:

    class TestKlasse
{
    public void Test(in Person a) {
        a.Navn = "";    // fejl
    }
}

struct Person {
    public string Navn;
}

    Hvis in benyttes ved referencebaserede variabler betyder det blot, at referencen ikke kan ændres.

    Expression-bodied members

    Lambda syntaks til:

    • methods
    • read only property
    • property
    • constructor
    • finalizer
    • indexer
    class Person
{
    // Egenskab
    private string _navn;
    public string Navn
    {
        get => _navn;
        set => _navn = value;
    }

    // Read only egenskab
    private int _alder;
    public int Alder => _alder;


    // Metoder
    public string NavnMedStort() => _navn?.ToUpper();
    public string ForBogstav(bool medStort) => medStort?_navn?.Substring(0,1).ToUpper(): _navn?.Substring(0, 1).ToLower();

    // Constructor
    public Person(string navn) => _navn = navn;
    // Finalizer
    ~Person() => Console.WriteLine("Oprydning");
}

public class SportsGrene
{
    private string[] typer = { "Håndbold", "Basketball", "Fodbold", "Tennis","Volleyball" };

    // Indexer
    public string this[int i]
    {
        get => typer[i];
        set => typer[i] = value;
    }
}

    Attributter

    For at give yderligere information til eksempelvis kompiler eller frameworks som benytter en metode kan den beriges med [attributter]. Det vil tilføjes ekstra metadata som kan aflæses hvis det ønskes. Eksempelvis kan en metode markeres med en ObsolteAttribute for at indikere, at metoden ikke bør benyttes:

    [Obsolete()]
public void Test(){}

// Prøv evt [Obsolete(true)]

    Nu vil eksempelvis Visual Studio markere metoden som obsolete (evt som fejl).

    Der må gerne tilføjes mange attributter og man kan skabe dem selv, men lige nu skal du bare vide at det er en måde at tilføje ekstra information til metoder.

    Introduktion til Extension methods

    Extension methods giver mulighed for at udvide eksisterende klasser med egne statiske metoder. Prøv eksempelvis

    using System;

namespace RetryPattern
{
    class Program
    {
        static void Main(string[] args)
        {
            // Array af tal
            int[] tal = { 5, 1, 5, 1, 6 };
            // tal.
        }
    }
}

    Prøv i VS eller VSC at skrive tal. og se listen af mulige metoder. Prøv herefter at tilføje LINQ

    using System;
using System.Linq;

namespace RetryPattern
{
    class Program
    {
        static void Main(string[] args)
        {
            // Array af tal
            int[] tal = { 5, 1, 5, 1, 6 };
            // tal.
        }
    }
}

    Prøv at skrive tal. igen og bemærk, at listen af muligheder er langt større. Array-klassen er udvidet med extension methods fra System.Linq.

    I mere avanceret C# kan du selv udvikle egne extension methods.

    Egne extension methods

    • Mulighed for at udvide eksisterende klasser med egne statiske metoder
      • Også selv det ikke er egne typer

    Hjælpebiblioteker

    • Skal være offentlige statiske metoder i en statisk klasse
    • Kan opfattes (og bruges) som et hjælpebibliotek
    • Første parameter skal være this
    • Må have så mange parametre man vil
    • For at extension metoder skal være tilgængelige skal der importeres det korrekte namespace
    using System;

namespace Extension
{
    public static class MineMetoder
    {
        // Udvider DateTime
        public static bool IsWeekend(this DateTime d) {
            if (d.DayOfWeek == DayOfWeek.Sunday || d.DayOfWeek == DayOfWeek.Saturday)
                return true;

            return false;
        }

        // Udvider string
        public static string LeftPartOfString(this string text, int number)
        {
            text += new string(' ', number);
            return text.Substring(0, number);
        }
    }
}

    var d1 = new DateTime(2018, 9, 1);
var d2 = new DateTime(2018, 9, 5);
Console.WriteLine($"d1.IsWeekend = {d1.IsWeekend()}");
Console.WriteLine($"d2.IsWeekend = {d2.IsWeekend()}");

string a = "123";
Console.WriteLine(a.Substring(0, 3));
// Console.WriteLine(a.Substring(0, 4));   // Fejl - streng ikke lang nok

// Denne virker
Console.WriteLine(a.LeftPartOfString(4));

// Kan jo også kaldes direkte - som et hjælpebibliotek
Console.WriteLine(Extension.MineMetoder.LeftPartOfString("123", 4));

/*
 ---------- Output: ----------

d1.IsWeekend = True
d2.IsWeekend = False
123
123
123

*/
