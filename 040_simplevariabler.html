    Simple variabler

    Grundlæggende om variabler

    Video Hukommelse (9 min.)

    Se video

    En C# app bliver automatisk tildelt et område i hukommelsen at boltre sig på - noget bliver brugt til det oversatte program (flow, metoder mv), og noget til midlertidige data

    • Stack
    • Heap

    Hukommelse er jo i realiteten en samling elektroniske komponenter (porte og kredsløb) der kan gemme tal som binære værdier.

    Variabler

    Alle applikationer har behov for at gemme midlertidige data, og i de fleste programmeringssprog sker dette ved hjælp af variabler.

    C# er typestærkt så kompileren ved præcis hvor meget der skal afsættes i hukommelsen. I stedet for at vi selv skal sørge for at allokere hukommelse, og efterfølgende holde styr på hvor i hukommelsen vi skal finde data, så stiller kompileren variabler til rådighed. I realiteten er det blot en tabel over nøgler (variabel navne) og adresser i hukommelsen samt allokeret plads.

    Typesystemet

    Video Typesystemet (3 min.)

    Se video

    Mange forskellige typer i .NET:

    • Heltal af forskellig størrelse
    • Reelle tal (kommatal) af forskellig størrelse
    • Dato og tid
    • Boolske værdier (sand/falsk)
    • Tegn eller tekster (også kaldet strenge)
    • Referencer til objekter eller strukturer

    Typerne er opdelt i

    • ValueTypes (værdibaserede typer)

      • Struct
        • bool (System.Boolean)
        • int (System.Int32)
        • float (System.Float)
        • decimal (System.Decimal)
        • DateTime (System.DateTime)
        • …

    • ReferenceTypes (referencebaserede typer)

      • Class
        • string (System.String)
        • arrays (System.Array)
        • lister (System.Collection.Generic....)
        • exceptions (System.Exception)
        • …

    Grundlæggende er forskellen på de to typer, at værdier fra værdibaserede variabler er placeret et sted i hukommelsen kaldet stak, og værdier fra de referencebaserede variabler er placeret et andet sted i hukommelse kaldet heap. Variabler af begge typer er placeret på stak’en men de værdibaserede typer indeholder konkrete værdier (ved heltal er værdien et konkret tal), medens variabler af referencebasrede typer indeholder referencer til et sted på heap’en. Meget mere om det senere.

    Erklæring af variabler

    Først angives typen

    • C# typenavn eller System-typenavn

    Herefter navnet på variablen

    [variabeltype] [navn]

    Husk - der er forskel på store og små bogstaver

    • A er forskellig fra a
    • navnPåPerson er forskellig fra navnpåperson

    Navngivning

    • Navnet skal begynde med et bogstav eller en underscore (_)
    • Navnet må ikke kun bestå af underscores
    • Navnet må kun bestå af bogstaver, tal eller underscores
    • Man må gerne på Æ Ø Å (men bør man?)

    Navngivningsstandard

    • Tal dansk eller engelsk?
    • Design Guidelines (http://goo.gl/YVxNz)
    • Brug eventuelt .editorconfig

    Eksempler på navngivning af lokale variabler

    int antal;
int antalMåneder;
int antalMånederPrÅr;
string navnPåPerson;

    Man kan vælge at bruge system navne (men best pratice er brug af genvejsnavne):

    System.Int32 antal;
System.Int32 antalMåneder;
System.Int32 antalMånederPrÅr;
System.String navnPåPerson;

    Variablen kan eventuelt tildeles en værdi ved erklæring

    int antal = 1;

// Brug evt systemnavn - giver det samme
// System.String navnPåPerson = "abc";

    Man kan vælge at benytte new ved oprettelse af simple variabler - men det er ikke nødvendigt ved de fleste simple værdibaserede variabler:

    int a;

    Virkefelter, Stack og Heap

    Video Virkefelter i C# (6 min.)

    Se video

    Når du ser et virkefelt i C# ({ }) skal du tegne en tegning i hovedet (Stack-definition).

    {
    // virkefelt
}

// betingelse
bool a = true;
if (a == true)
{
    // virkefelt
}

// løkke
for (int i = 0; i < 4; i++)
{
    // Virkefelt
}

// metode
void Test()
{
    // Virkefelt
}

    Den fortæller hvilke variabler der er tilgængelige i det givne virkefelt, og hvilken værdier (eller reference) disse variabler har.

    Hvis et virkefelt er placeret i et andet virkefelt må du tegne det som en stack i en stack

    Indre virkefelter har adgang til variabler i de ydre virkefelter - ikke omvendt (og ikke virkefelter på samme niveau).

    int a = 1;
Console.WriteLine($"a = {a}");

{
    int b = 2;
    Console.WriteLine($"a = {a}");
    Console.WriteLine($"b = {b}");
    {
        int c = 3;
        Console.WriteLine($"a = {a}");
        Console.WriteLine($"b = {b}");
        Console.WriteLine($"c = {c}");
    }
}

    De fleste simple variabeltyper beholder værdien på stacken, men enkelte typer placerer værdien andre steder i hukommelsen (den såkaldte heap), og variablens værdi er så en reference til et andet sted i hukommelsen. Mere om det senere!

    double / decimal

    Pas lige på med double (64 bit) - typen er en base2 type og benytter floating point hvilke kan give nogle afrundningsfejl.

    {
    double v = .1;
    double r = v + v + v + v + v + v + v + v + v + v;
    // r = 0.99999999999999989
}
{
    decimal v = .1M;
    decimal r = v + v + v + v + v + v + v + v + v + v;
    // r = 1
}

    Til gengæld er den vanvittig hurtig - som i 20 gange hurtigere end decimal (kode fra exceptionnotfound):

    private static int iterations = 100000000;

private static void DoubleTest()
{
    Stopwatch watch = new Stopwatch();
    watch.Start();
    Double z = 0;
    for (int i = 0; i < iterations; i++)
    {
        Double x = i;
        Double y = x * i;
        z += y;
    }
    watch.Stop();
    Console.WriteLine("Double: " + watch.ElapsedTicks);
    Console.WriteLine(z);
}

private static void DecimalTest()
{
    Stopwatch watch = new Stopwatch();
    watch.Start();
    Decimal z = 0;
    for (int i = 0; i < iterations; i++)
    {
        Decimal x = i;
        Decimal y = x * i;
        z += y;
    }
    watch.Stop();
    Console.WriteLine("Decimal: " + watch.ElapsedTicks);
    Console.WriteLine(z);
}

    Overflow

    Brug checked hvis overflow skal skabe exception

    {
    byte b = 255;
    Console.WriteLine($"b={b}");
    b++;
    Console.WriteLine($"b={b}");
}

checked
{
    byte b = 255;
    Console.WriteLine($"b={b}");
    // b++; skaber et overflow
}

    Lidt mere avanceret

    De fleste simple typer indeholder også en [type].Parse:

    string a = "434";
int b = int.Parse(a);
double c = double.Parse(a);

    Konvertering en af tekst som indeholder en dato kan være ret kompleks. Derfor indeholder DateTime-typen en statisk metode ParseExact, som kan være brugbar. Den konverterer en streng til en dato ved hjælp af en skabelon og en evt kultur (typisk bruge CultureInfo.InvariantCulture (format er ikke er bundet til en kultur), men man kan benytte en specifik kultur).

    using System.Globalization;

string tekst, format;
DateTime res;

tekst = "2022|07";
format = "yyyy|MM";
res = DateTime.ParseExact(tekst, format, CultureInfo.InvariantCulture);
Console.WriteLine(res);

tekst = "10. juli 2022";
format = "D";
res = DateTime.ParseExact(tekst, format, new CultureInfo("da-DK"));
Console.WriteLine(res);

    Alternativt kan man i nogen sammenhæng benytte ([type]) ved konvertering - men pas på ved tal. Benyttes mest ved referencebaserede variabler:

    double a = 4030.75;
int b = (int)a;  // 4.030 -- PAS PÅ
