    Om modulet

    Helt grundlæggende om C#-programmering

    • Mange indbyggede simple datatyper
      • variabler
    • BCL (Base Class Library) stiller en masse funktionalitet til rådighed
      • System.xxx

    • Alt er baseret på typer

      • I den grundlæggende C#
        • Klasser
        • Strukturer
        • Enumerations
        • Delegates
        • Interface

    • I grundlæggende C# behøver man ikke være særlig bevidst eller bekymret for oprydning af hukommelse

      • Modsat andre sprog som eksempelvis C eller C++

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            MinTæller tæller = new MinTæller(5, ConsoleColor.Red);
            tæller.Tæl();
        }

    }

    class MinTæller {
        public int Antal { get; private set; }
        public ConsoleColor FarveTilSøndag { get; private set; }

        public MinTæller(int antal, ConsoleColor farve)
        {
            this.Antal = antal;
            this.FarveTilSøndag = farve;
        }

        public void Tæl() {
            Console.WriteLine($"Starter med at tælle");
            for (int i = 0; i < this.Antal; i++)
            {
                if (DateTime.Now.DayOfWeek == DayOfWeek.Sunday)
                {
                    Console.ForegroundColor = this.FarveTilSøndag;
                }
                else
                {
                    Console.ForegroundColor = ConsoleColor.Gray;
                }
                Console.WriteLine(i);
            }
            Console.WriteLine("Færdig med at tælle");
        }
    }
}

    C# er funktionsorienteret

    Video C# er funktionsorienteret (4 min.)

    Se video

    using System;
using System.Collections.Generic;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            Tæl(Enumerable.Range(0, 5),
                () =>
                {
                    if (DateTime.Now.DayOfWeek == DayOfWeek.Sunday)
                        Console.ForegroundColor = ConsoleColor.Red;
                },
                () => Console.ForegroundColor = ConsoleColor.Gray,
                Console.WriteLine);


            // Blot eksempel på kald af funktionen med en anden
            // nummerrække og referencer til andre skriv/farve funktioner
            Tæl(Enumerable.Range(1, 10).Where(i => i % 2 == 0),
                () =>
                {
                    if (DateTime.Now.DayOfWeek == DayOfWeek.Sunday)
                        System.IO.File.AppendAllText(@"c:\temp\side.html", "<div class='red'>\r\n");
                },
                () => System.IO.File.AppendAllText(@"c:\temp\side.html", "<div class='gray'>\r\n"),
                (t) => System.IO.File.AppendAllText(@"c:\temp\side.html", "\r\n" + t)
            );


        }

        static void Tæl(IEnumerable<int> tal,
          Action metodeTilAtSætteFarve,
          Action metodeTilAtResetFarve,
          Action<string> metodeTilAtSkrive)
        {
            metodeTilAtSkrive($"Starter med at tælle");
            tal.ToList().ForEach((i) =>
            {
                metodeTilAtSætteFarve();
                metodeTilAtSkrive(i.ToString());
                metodeTilAtResetFarve();
            });
            metodeTilAtSkrive("Færdig med at tælle");
        }
    }
}

    Alt er baseret på typer

    Video Alt er baseret på typer (6 min.)

    Se video

    I C# er alt baseret på nogle få typer

    • class (klasse)
    • struct (struktur)
    • interface (definitation af medlemmer på en type)
    • delegate (“funktionspointer”)
    • enum (relateret konstant)
    • records (autogenereret klasse/struct - ny i C# 9 og 10)

    I den helt grundlæggende C# kan du nøjes med at fokusere på at skabe typer baseret class og struct - og i de fleste tilfælde blot på class.

    Men både struct og class (og record) skal ses som en skabelon for instanser

    Instanser af klasser kaldes typisk for objekter.

    I disse instanser er der typisk både data (tal, tegn, datoer, referencer til andre objekter), samt metoder der kan arbejde med disse data.

    namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // a er en instans af System.Int32 (struct) og kan opbevare en heltalsværdi (32 bit)
            System.Int32 a = new System.Int32();
            a = 10;

            // kan også skrives nemmere som
            // int a;
            // a = 10;
            // eller
            // int a = 10;

            // System.Int32 indeholder metoden ToString(), der konverterer heltallet til en streng
            string b = a.ToString();

            // ------------------------

            // c er en instans af System.DateTime (struct) og kan opbevare dato og tid
            System.DateTime c = new System.DateTime();
            c = System.DateTime.Now;

            // kan også skrives som
            // System.DateTime c;
            // c = System.DateTime.Now;    // System tid
            // eller
            // System.DateTime c = System.DateTime.Now;

            // System.DateTime indeholder en masse metoder der kan bruges på den gemte dato
            System.Console.WriteLine(c.AddDays(1));
            System.Console.WriteLine(c.IsDaylightSavingTime());

            // System.Random (class) kan bruges til at finde et tilfældigt tal
            System.Random d = new System.Random();
            // og indeholder en masse metoder relateret til dette
            System.Console.WriteLine(d.Next(1, 10));

            // Hund (class) er type med data (navn og antalben) og en enkelt metode
            Hund e = new Hund();
            e.Navn = "Fido";
            e.AntalBen = 4;
            System.Console.WriteLine(e.SigNoget());


            // Både System.Int32, System.DateTime, System.Random og Hund er typer (struct eller class)

        }

    }

    class Hund
    {
        public string Navn;
        public int AntalBen;
        public string SigNoget()
        {
            return $"Jeg hedder {this.Navn} og har {this.AntalBen} ben.";
        }
    }
}

    Namespace

    Video Namespace (7 min.)

    Se video

    Alle typer er placeret i et namespace. Et namespace kan bestå af typer samt andre namespace’s, og kan man derfor opbygge et hieraki af typer. Det sikrer logisk struktur og indkapsling, og samtidigt undgår man evt navnesammenfald.

    Det bedste eksempel er MS egen BCL hvor alt er placeret i eller under System-namespacet

    Her er blot nogle få eksempler under System

    • System.Console
    • System.IO
    • System.Data
    • System.Math
    • System.Net
    • System.Xml
    • System.Collections
    • System.Diagnostics
    • System.Reflection
    • System.Security
    • System.Threading
    • System.Random

    Man kan selv erklære et namespace hieraki, og her opbevare forskellige typer som

    • klasser
    • strukturer
    • enumerations
    • delegates

    namespace NS1
{
    public class A { }
    public class B { }
    public struct C { }
    public enum D { }
    public delegate Action E();
}

    Et namespace omkranser typer og sikre struktur og forhindre navnesammenfald

    namespace NS1
{
    public class A { }
    public class B { }
    public struct C { }
    public enum D { }
    public delegate Action E();
}

namespace NS2
{
    public class A { }
    public class B { }
    public struct C { }
    public enum D { }
    public delegate Action E();
}

    Et namespace må gerne være i sit eget namespace (som ligger i sit eget namespace (som …))

    namespace NS1
{
    namespace NS2
    {
        public class A { }
        public class B { }
        public struct C { }
        public enum D { }
        public delegate Action E();
    }

    namespace NS3
    {
        public class A { }
        public class B { }
        public struct C { }
        public enum D { }
        public delegate Action E();
    }
}

    Tilgang til typer

    Video Tilgang til typer (1 min.)

    Se video

    Ved definering af typer angives også en access modifier. Det er besked til compileren om hvor typen kan benyttes.

    På namespace-niveau

    I grundlæggende C# benyttes to

    • internal (privat - kun tilgængelig i projekt)
    • public (offentlig - alle projekter med reference kan tilgå typen)

    Begyndere bør angive en access modifier - men internal er default

    public class A
{
}

internal class B
{
}

class C // internal som default
{
}

    Tilgang til typer på type niveau

    I grundlæggende C# benyttes to

    • private (privat - kun tilgængelig i typen)
    • public (offentlig - alle kan tilgå typen)

    Men igen - typer i typer er ikke så meget benyttet i den grundlæggende C#.

    public class A
{
    private class B
    {

    }
