    Indkapsling

    Om modulet

    I dette modul vil du lære om indkapsling og beskyttelse af data. Vi skal både se på brugen af de klassiske get/set metoder, samt brugen af egenskaber som i realiteten blot er autogenerering af metoder. Sidstnævnte er dog måden at indkapsle data på i C#, og du vil forstå hvorfor det er at foretrække.

    Grundlæggende om indkapsling

    Video Grundlæggende om indkapsling (5 min.)

    Se video

    Brug af synlighed (private/public) til at indkapsle data og skabe adgang ved hjælp metoder eller egenskaber.

    Grundlæggende kan der i C# vælges mellem to måde at indkapsle data på

    • metoder
    • egenskaber
    • som egentlig bare er autogereret kode der benytter metoder

    Indkapsling med egenskaber

    Video Indkapsling med egenskaber (15 min.)

    Se video

    • Brug af egenskaber i stedet for manuelle get/set metoder er bedre fordi

    • Syntaks er langt bedre

    • Klar typemæssig adskillelse mellem medlemmer som “udelukkende” arbejder på data og som medlemmer som er metoder
      • Eksempelvis kan brugerflader som WPF/WinForm bruge det til databinding fordi man gennem kode kan se om der er tale om et medlem som har med objektets data at gøre
    • get/set metoder autogenereres
    using System;

namespace Klasser9
{
    public class Person
    {
        private string navn;

        public string Navn
        {
            get
            {
                // måske kode som håndterer:
                // log
                // sikkerhed

            }
            set
            {
                // måske kode som håndterer:
                // log
                // validering
                // sikkerhed

            }
        }

        private int alder;

        public int Alder
        {
            get
            {
                // måske kode som håndterer:
                // log
                // sikkerhed

            set

}

    init

    I C# 9 blev det muligt at skabe read-only lignende egenskaber med init-kodeordet:

    public class Person
{
    private string? navn;

    public string? Navn
    {
        get { return navn; }
        init { navn = value; }
    }

    private int alder;

    public int Alder
    {
        get { return alder; }
        init { alder = value; }
    }
}

    Nu kan egenskaberne udelukkende tildeles en værdi ved initialisering:

    Person person = new Person() { Navn = "a", Alder = 10 };
// person.Navn = "b";  // fejl

    Automatiske egenskaber

    Video Automatiske egenskaber (8 min.)

    Se video

    • Hvis man blot vil sende værdier videre til felter kan man benytte automatiske egenskaber
    • Man kan evt senere ændre det til en fuld egenskab med kode til sikkerhed, log mv
    using System;

namespace Klasser11
{
    public class Person
    {

        // fuld egenskab
        private string navn;

        public string Navn
        {
            get { return navn; }
            set { navn = value; }
        }

        // forskellige automatiske egenskaber
        public int Alder { get; set; }
        public string Land { get; set; } = "";
        public double Løn  { get; }
        public bool HeltSpecielPerson { get; private set; } // kan kun tildeles internt
        public string Adresse { get; init } // kan kun tildeles ved initialisering

    }
}

    Autogenerering af kode

    Brug eventuelt NuGet-pakken MCronberg.CSCourseHelper.Standard (søg efter “cronberg”). Den indeholder blandt andet en Extension-metode til Object kaldet ToStringEx, som giver flere informationer om et objekt. Så kan man eksempelvis nemt se, at en fuld egenskab og automatisk egenskab resulterer i autogenerede felter og metoder:

    using System;
using MCronberg;

namespace demo
{
    class Program
    {
        static void Main(string[] args)
        {
            Dyr d = new Dyr();
            Console.WriteLine(d.ToStringEx());
        }
    }
    class Dyr
    {
        // Automatisk egenskab
        public int Id { get; set; }

        // Fuld egenskab
        private string navn;

        public string Navn
        {
            get { return navn; }
            set { navn = value; }
        }
    }
}

````
